[PROJECT] SyncNyang — Godot 4.x 리듬게임 심각한 렌더링/판정 버그 진단 요청

현재 Godot 4.x (GDScript) 기반 모바일 리듬게임을 개발 중인데, UI/UX 리팩토링 후 아래와 같이 완전히 게임이 망가지는 현상이 발생했습니다. 원인과 해결책을 분석해 주십시오.

### [발생 중인 심각한 3가지 버그]
1. **레일(Lane) UI 증발:** 3개의 레일에 `TextureRect`를 적용했는데, 화면에서 완전히 사라지고 아예 보이지 않습니다. 
2. **노트 역주행:** 화면 위에서 아래(HitLine)로 떨어져야 할 노트(Dot)들이 반대로 "위로 올라가는(역주행)" 현상이 발생합니다.
3. **판정 및 피버 먹통:** 노트가 닿았을 때 터치해도 판정이 전혀 되지 않아 전부 MISS가 뜨며, 피버 게이지(Fever Gauge) 역시 그래픽적으로 전혀 차지 않습니다.

### [관련 구현 방식 요약]
- **노트 위치 계산 (GameController):**
  노트의 현재 Y 좌표는 `lerpf(spawn_y, hitline_y, progress)` 로 계산됩니다. `spawn_y`는 200.0, `hitline_y`는 게임 시작 시 `game_ui`에서 `hit_line.position.y` 값을 넘겨받습니다. `hit_line.position.y`는 `화면_전체_높이 - 하단여백 - 250.0` 으로 코드로 설정합니다.
- **레일 구현 (GameUI):**
  `TextureRect.new()`로 3개의 레일을 코드로 동적 생성 후 `add_child` 합니다. 깊이 정렬을 위해 `move_child(rect, hit_line.get_index() + 1)`을 사용하고, 뷰포트 resize 시점에 크기를 재계산해 강제 주입합니다.
- **입력 판정 (GameUI -> GameController):**
  화면 전체를 덮는 `TapArea` (Panel)의 `gui_input` 이벤트를 받아, 터치 좌표 `event.position`을 보정하여 눌린 레일을 찾습니다. 
- **피버 게이지:**
  점수에 따라 시그널이 오면 `_gauge_fill.size.x = 최대넓이 * clampf(value / 100.0, 0, 1)` 로 ColorRect의 width를 조절합니다.

### [질문 요건]
1. **노트 역주행:** 위에서 아래로 내려와야 할 노트가 위로 올라간다면, `game_ui` 측에서 `hitline_y` (목표점) 값이 `spawn_y`(200.0)보다 작게(음수 또는 더 위쪽으로) 측정되었다는 뜻인데, Godot 4.x UI 시스템의 Anchor/Position 초기화 타이밍에서 어떤 치명적 오류가 있었을까요?
2. **레일 증발:** 코드로 동적 생성한 `TextureRect`가 화면에 나타나지 않거나 크기가 0이 되어버리는데, 앵커 충돌 현상이나 부모 컨테이너 설정 누락 관점에서 진단해주세요.
3. **터치 무시 및 판정 먹통:** `TapArea` (Panel)에서 터치를 못 잡거나 좌표가 엉뚱한 곳으로 찍히는 뷰포트 배율(Stretch mode) 문제나 레이어 겹침 문제가 의심됩니다. 어떻게 수정해야 할까요?

이 현상(노트 역주행 + 텍스처 투명화)의 근본 원인 1~2가지를 짚어주세요.
